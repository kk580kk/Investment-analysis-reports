**完成后更新：** `- [x] 1. 需求澄清`

## 根据研究类型调整流程

在步骤1明确研究目标后，根据用户需求调整执行深度：

| 类型 | 研究维度 | 来源数 | 报告长度 | 触发词示例 |
|------|----------|--------|----------|------------|
| 快速概览 | 3-5个 | 10-15个 | ~2000字 | "快速了解"、"简单调研"、"大概看一下" |
| 标准研究 | 5-8个 | 20-30个 | ~5000字 | 默认模式 |

## 步骤 2：研究规划

**目标：** 构建研究框架，规划搜索策略

**操作：**
1. 拆分主题为5-8个核心研究维度
2. 为每个维度设计2-3个搜索关键词变体
3. 确定信息源优先级

**研究维度模板：**
```
├─ 📊 现状分析：当前状态如何？核心数据？
├─ 🔍 技术原理：底层机制/技术是什么？
├─ 🏢 主要玩家：谁是关键参与者？
├─ 📈 趋势分析：发展方向？未来预测？
├─ ⚖️ 挑战与风险：面临哪些问题？
├─ 💡 机遇与前景：有哪些潜在机会？
└─ 🌍 区域差异：不同地区/市场的差异？
```

**完成后更新：** `- [x] 2. 研究规划`

## 步骤 3：信息搜索与提取

**目标：** 使用sub-agents并发执行搜索，并提取关键信息进行返回

### 3.1 启动并发研究Sub-Agent

为每个研究维度启动独立的sub-agent，使用Agent工具并发执行。

### 3.2 Sub-Agent任务规范

**⚠️ 关键：Sub-Agent 不只是执行搜索命令，必须主动完成信息提取全流程：**

1. **执行搜索** → 调用搜索脚本获取结果列表
2. **分析结果** → 从搜索结果中筛选2-3个高价值来源（优先学术/官方来源）
3. **深度获取** → 访问并阅读筛选出的来源内容
4. **提取信息** → 从来源中提取：核心数据、关键观点、利益相关者、争议点
5. **结构化输出** → 返回JSON格式的结构化结果

**Sub-Agent Prompt模板：**
```
你是研究[dimensions]的专家agent。请完成以下任务：

1. 针对关键词"[search_keywords]"执行搜索，使用如下搜索命令进行信息搜索
  ```bash
  node /home/sandbox/.openclaw/workspace/skills/xiaoyi-web-search/scripts/search.js "[关键词]" -n 8
  ```
1. 分析搜索结果，筛选2-3个高价值来源（优先学术论文、官方报告、权威媒体）
2. 自主决定是否使用WebFetch工具深度获取这些来源的内容
3. 从内容中提取关键信息：核心数据、关键观点、主要参与者、争议点
4. 返回结构化JSON结果

禁止只返回搜索结果列表，必须提取并结构化信息。
```

**输出格式：**
```json
{
  "维度": "[维度名称]",
  "关键发现": [{"发现": "...", "数据": "...", "来源": "url", "置信度": "高/中/低"}],
  "核心数据": [...],
  "来源列表": [{"标题": "...", "url": "...", "可信度": "⭐⭐⭐"}]
}
```

### 3.3 并发执行示例

使用 `sessions_spawn` 工具并发启动多个话题研究的 sub-agents：

```
# 在单条消息中并发启动多个SubAgent
SubAgent(dimensions="现状分析", search_keywords=["[主题] 现状 概述", "[主题] 市场 报告"])       
SubAgent(dimensions="技术原理", search_keywords=["[主题] 技术 原理", "[主题] 架构 实现"])       
SubAgent(dimensions="市场格局", search_keywords=["[主题] 市场 规模", "[主题] 竞争 格局"])       
SubAgent(dimensions="挑战风险", search_keywords=["[主题] 挑战 问题", "[主题] 风险 困难"])       
SubAgent(dimensions="趋势预测", search_keywords=["[主题] 趋势 未来", "[主题] 发展 预测"]) 
```

## 步骤 4：交叉验证

**目标：** 验证关键数据，识别矛盾

**操作：**
1. 对关键数据点进行跨维度验证
2. 构建验证矩阵
3. 标注数据置信度

**验证矩阵格式：**
| 数据项 | 来源A | 来源B | 验证结果 |
|--------|-------|-------|----------|
| 市场规模 | $X亿 | $Y亿 | ✅一致 / ⚠️差异 |

**完成后更新：** `- [x] 4. 交叉验证`

## 步骤 5：深度分析

**目标：** 综合信息，形成洞察

- 风险评估：技术/市场/政策风险

## 步骤 7：质量审查

**目标：** 确保报告质量

**检查清单：**
- [ ] 所有数据点有来源引用
- [ ] 优先使用最近12个月数据
- [ ] 呈现不同观点，不偏颇
- [ ] 覆盖所有研究维度
- [ ] 结构清晰，层次分明

**审查循环：**
1. 执行上述checklist检查
2. 发现问题 → 标注问题项，返回对应步骤修复
3. 修复后重新检查
4. 循环直到全部通过
