## 步骤一：信息整理

**目标**：收集足够的信息用于生成 PPT 内容。

### 1.1 判断信息来源

分析用户请求，按以下优先级确定信息来源：

**A. 用户提供了文档**（本地文件路径、附件、上传的文件等）：

1. 查找已安装的文档解析 skill（名称含 `doc`、`document`、`parse`、`reader` 等）
2. 使用该 skill 解析用户指定的全部文档，提取完整内容
3. 整理提取的关键信息、数据、结构和图表

**B. 用户未提供文档，或 Agent 判断现有信息不足以生成高质量 PPT**：

1. 查找已安装的网络搜索 skill（名称含 `search`、`web`、`browse` 等）
2. 根据用户需求和主题，拟定 3-5 个有针对性的搜索关键词
3. 执行搜索，汇总相关信息
4. 如信息仍不充分，可进行 2-3 轮补充搜索

> 两种来源可以并用：先解析文档，再用网络搜索补充文档中缺少的数据或背景信息。

### 1.2 信息整合

整理所有收集到的信息，识别：
- 核心主题和关键论点
- 支撑数据、案例、图表
- 逻辑结构和层次关系

**完成汇报**：
```
✅ 信息整理完成，共收集 X 个关键信息点
```

## 步骤二：梳理写作思路

**目标**：基于步骤一收集的信息，提炼出一份**简洁的写作思路**，包含以下要点：

- **主题与核心立意**：PPT 要讲什么、核心观点或目的是什么
- **整体逻辑框架**：高层次的叙述脉络（如：现状 → 问题 → 方案 → 效果），不列具体章节标题
- **确定风格主题**：从下表中选一，给出 key 值并说明理由

  | 主题 | key | 适用场景 |
  |------|-----|---------|
  | 学术 | `academic` | 研究报告、论文答辩 |
  | 商务 | `business` | 商业提案、年度汇报 |
  | 科技 | `tech` | 技术方案、产品发布 |
  | 国风 | `ink` | 传统文化、古典文学 |
  | 党政 | `politics` | 党建工作、政策宣讲 |
  | 简约 | `minimal` | 通用场景、教学演示 |

- **确定结构类型与目标页数**：

  | 条件 | 结构类型 |
  |------|---------|
  | 内容页 ≤12 | 单层结构（封面 → 目录 → 内容页） |
  | 内容页 >12 | 双层结构（封面 → 目录 → 章节页 → 内容页） |

  硬性约束：单层内容页总数 ≤8；双层章节数 ≤8，每章内容页 ≤8

  页数默认规则（用户未指定时）：
  - 内容较少或主题简单：8～10 页内容页
  - 内容适中（常规汇报/方案）：10～15 页内容页
  - 内容丰富（详细报告/研究）：15～20 页内容页
  - 用户明确指定页数时，以用户要求为准

> 写作思路应简洁（5-8 条），聚焦方向和逻辑，**不列具体页面标题或内容细节**。写作思路中必须包含明确的风格 key、结构类型和目标页数，以便用户一并确认。

### A.1 读取生成指南

### A.1.5 图片预处理

如果用户提供了图片文件（照片、截图等需要插入 PPT 的图片），在生成前统一转换格式：

```bash
python3 ~/.openclaw/workspace/skills/xiaoyi-ppt/scripts/normalize_images.py image_path
```

> iPhone 拍摄的 `.jpg` 实际可能是 MPO（多帧）格式，python-pptx / PptxGenJS 直接嵌入会导致显示异常。此步骤通过 Pillow 统一转为标准 JPEG，成本极低，无需判断格式。

### A.2 生成 PPT

基于步骤二的写作思路，**一步到位**生成完整 `.pptx` 文件：

**结构**：封面 → 目录（页数 ≥8 时加入）→ 内容页 → 结束页。每页聚焦一个要点。

**设计基本要求**：
- 自行选定一组与主题匹配的主色 / 辅色 / 强调色（6 位 hex，无 `#`）
- 封面和结束页用深色背景，内容页用浅色背景
- 每页至少包含一个视觉元素（色块、形状、图标），避免纯文字页
- 相邻页不重复同一版式
- 标题 36-44pt，正文 14-16pt
- **页面类型组成**：图文页（text_image）≥ 内容页 25%；纯数据页（stats + big_number）合计 ≤3 页；禁止连续 2 个 stats/big_number。数据优先用 cards 或 numbered_list 承载

**输出**：
```bash
$PPT_SESSION_DIR/output.pptx
```
